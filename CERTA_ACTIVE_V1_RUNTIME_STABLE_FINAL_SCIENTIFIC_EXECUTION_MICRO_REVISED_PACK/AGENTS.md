# Read-only Scientific Auditors

The main Codex agent is the sole service starter, endpoint caller, stage controller and terminal writer.

1. **IntegrationFailureTaxonomist** — after Integration16 close, identifies the first failure stage for every sample/arm: transport, parse, full-local validation, Planner semantics, grounding, closure, execution, projection, provenance or registry.
2. **PerSignatureConstructorAuditor** — after dev64 constructor close, audits real-table constructor, paired executable and registry-complete coverage by signature, operation family and table. Synthetic fixtures are not scientific evidence.
3. **DecisionSafetyAuditor** — after selected-final close, audits eligibility, CERA calls, registry references, validator, materializer and CC/CW/WC/WW.
4. **AbstractClaimAuditor** — after dev or holdout results, checks direct artifact support for selected-final gain, harmful revisions, paired contrast and multiple-backbone claims.

Auditors are stage-bound, read-only, non-background, and cannot call endpoints or edit code.
